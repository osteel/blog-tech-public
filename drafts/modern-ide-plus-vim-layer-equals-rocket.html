How do you generate a random string? Open a Vim editor, and try to exit!

To a lot of people, this is what their experience of Vim looks like: some trying-to-exit Vim jokes, and sights of relief when they realise Nano is available on the system they're interacting with.

Besides the fact that Nano is at least equally weird to exit to me, this reputation of being complicated precedes Vim in a way that drives most people away before they even tried, thus missing out on its great potential.

This is especially true for programmers.

I am not necessarily talking about using Vim as full-fledged IDE, which is an approach I once tried before deciding that some pains are not worth living through (I may upset a few Vim purists here, although they are probably too busy fighting some Emacs-imalists somewhere).

I'm talking about adding a Vim layer to your favourite IDE, which probably already exists as an extension: just type "Vim" in your extension manager and I'm sure you will get some results (and if your IDE does not support extensions, it might be time to consider a change).

Here are some extensions for VSCode, Sublime, PHPStorm, and Atom.

But beyond this apparent complexity and occasional turf squabbles lies an important fact: Vim's approach to text editing is powerful in a way that can benefit most programmers.

This post is not about how to use Vim, but a collection of examples that will hopefully make you want to dig further, and maybe start using it on a daily basis as well.

If that's the case, you will find links to various learning resources at the end of the article.
